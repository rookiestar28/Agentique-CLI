# Security Policy

Do not open public issues for vulnerabilities, abuse reports, private user data, credentials, or exploit details.

Report security concerns through the private disclosure channel selected by the Agentique owner before public launch. If no channel has been published yet, do not disclose details publicly.

This documentation repository cannot approve, certify, moderate, or publish Agentique resources. Platform decisions remain on `agentique.io`.

## Public Tracker Boundary

Public issues are not suitable for vulnerability details, exploit steps, abuse reports, credentials, private account data, personal data, moderation material, or unsafe resource contents.

If a report cannot be described without those details, use the private disclosure route selected by the Agentique owner or the platform-owned report/support flow.
